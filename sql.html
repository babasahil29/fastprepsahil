Language Snapshot

Introduction to SQL

  • Definition: SQL (Structured Query Language) is a standardized programming language specifically designed for managing, querying, and manipulating databases.
  • Creator: IBM
  • Year Created: 1970s
  • Purpose: SQL is essential for tasks like retrieving data, inserting new records, updating or deleting existing data, and defining database structures.

Key Features

  • Data Retrieval: SQL is widely used to query and extract data from relational databases using SELECT statements.
  • Data Manipulation: SQL allows inserting (INSERT), updating (UPDATE), and deleting (DELETE) records from databases.
  • Data Definition: SQL helps in creating and modifying database schemas and objects using Data Definition Language (DDL) commands like CREATE, ALTER, and DROP.
  • Data Control: SQL manages user access to databases using GRANT and REVOKE commands.
  • ACID Compliance: SQL databases adhere to the principles of Atomicity, Consistency, Isolation, and Durability, ensuring reliable transactions.

SQL Command Categories

  • Data Query Language (DQL):
    • SELECT: Retrieves data from the database.
  • Data Manipulation Language (DML):
    • INSERT, UPDATE, DELETE: Used for managing data within tables.
  • Data Definition Language (DDL):
    • CREATE, ALTER, DROP, TRUNCATE: Defines the structure and schema of databases and tables.
  • Data Control Language (DCL):
    • GRANT, REVOKE: Manages permissions and access control in databases.
  • Transaction Control Language (TCL):
    • COMMIT, ROLLBACK, SAVEPOINT: Used to manage transactions within databases.

Common Use Cases

  • Data Retrieval: Extracting information from large databases for reporting, analytics, or application use.
  • Database Administration: Maintaining database structures, users, permissions, and optimization.
  • Data Integration: Combining data from multiple tables or databases for processing.
  • Data Integrity & Security: Ensuring that data is accurate and access is controlled.

Relational Databases Using SQL

  • MySQL: Open-source and widely used in web applications.
  • PostgreSQL: Known for its advanced features and compliance with SQL standards.
  • SQLite: Lightweight, serverless database, often used in mobile and embedded systems.
  • Oracle: A popular RDBMS for enterprise-level applications.
  • Microsoft SQL Server: SQL implementation by Microsoft for enterprise-level applications.

Key SQL Terms

  • Table: A collection of rows and columns that stores structured data.
  • Primary Key: A unique identifier for each record in a table.
  • Foreign Key: A column that establishes a relationship between two tables.
  • Join: A method to retrieve data from multiple tables based on a related column.
  • Index: A database optimization feature to improve the speed of data retrieval.

History of SQL

  • 1970s: IBM develops SQL based on Dr. Edgar F. Codd's relational model.
  • 1986: SQL becomes an ANSI standard.
  • 1989: SQL becomes an ISO standard, cementing its use across relational databases globally.

Cheat Sheet

Easy Level

1. Select Data from Table:

Retrieves all data from the specified table.

            SELECT * FROM table_name;

2. Select Specific Columns:

Retrieves specific columns from the table.

            SELECT column1, column2 FROM table_name;

3. Insert Data into Table:

Inserts new data into the table.

            INSERT INTO table_name (column1, column2) VALUES (value1, value2);

4. Update Existing Data:

Updates data in a table based on a condition.

            UPDATE table_name SET column1 = value1 WHERE condition;

5. Delete Data:

Deletes data that meets the condition.

            DELETE FROM table_name WHERE condition;

6. Create a Table:

Creates a new table with specified columns and data types.

            CREATE TABLE table_name (
               column1 datatype,
               column2 datatype
            );

7. Drop a Table:

Deletes a table and all its data.

            DROP TABLE table_name;

8. Filtering Data with WHERE Clause:

Filters records based on specific criteria.

            SELECT * FROM table_name WHERE condition;

9. Sorting Data:

Sorts data in ascending or descending order.

            SELECT * FROM table_name ORDER BY column_name ASC|DESC;

10. Count Rows:

Returns the number of rows in the table.

            SELECT COUNT(*) FROM table_name;

Intermediate Level

1. Join Two Tables:

Joins two tables based on a common column.

            SELECT column_name(s)
            FROM table1
            INNER JOIN table2
            ON table1.column = table2.column;

2. Group Data:

Groups data by column and performs an aggregate function.

            SELECT column_name, COUNT(*)
            FROM table_name
            GROUP BY column_name;

3. Filter Grouped Data (HAVING):

Filters grouped data using aggregate conditions.

            SELECT column_name, COUNT(*)
            FROM table_name
            GROUP BY column_name
            HAVING COUNT(*) > 1;

4. Using Aliases:

Assigns temporary names (aliases) to columns.

            SELECT column_name AS alias_name
            FROM table_name;

5. Using IN Operator:

Filters records where the column value matches one of the values.

            SELECT * FROM table_name
            WHERE column_name IN (value1, value2, value3);

6. Subqueries:

Uses a subquery to filter data.

            SELECT * FROM table_name
            WHERE column_name = (SELECT column_name FROM another_table WHERE condition);

7. Create an Index:

Creates an index on a table to improve query performance.

            CREATE INDEX index_name ON table_name (column_name);

8. Distinct Values:

Selects only distinct (unique) values from a column.

            SELECT DISTINCT column_name FROM table_name;

9. Limit the Number of Results:

Limits the result set to a specific number of rows.

            SELECT * FROM table_name LIMIT 10;

10. Using LIKE for Pattern Matching:

Searches for a specified pattern in a column.

            SELECT * FROM table_name
            WHERE column_name LIKE 'pattern%';

Expert Level

1. Transactions:

Manages transactions by grouping multiple SQL operations.

            BEGIN;
            -- SQL operations
            COMMIT;

2. Rollback a Transaction:

Reverts changes made during a transaction.

            ROLLBACK;

3. UNION Operator:

Combines the result sets of two queries without duplicates.

            SELECT column_name FROM table_name1
            UNION
            SELECT column_name FROM table_name2;

4. FULL OUTER JOIN:

Returns all rows from both tables, with matching records from both sides.

            SELECT column_name(s)
            FROM table1
            FULL OUTER JOIN table2
            ON table1.column = table2.column;

5. Self Join:

Joins a table with itself.

            SELECT a.column_name, b.column_name
            FROM table_name a, table_name b
            WHERE a.common_field = b.common_field;

6. Create a View:

Creates a virtual table (view) based on a SQL query.

            CREATE VIEW view_name AS
            SELECT column_name(s) FROM table_name WHERE condition;

7. Using EXISTS:

Returns true if the subquery returns any records.

            SELECT * FROM table_name
            WHERE EXISTS (SELECT 1 FROM another_table WHERE condition);

8. Window Functions:

Applies a function across a set of table rows related to the current row.

            SELECT column_name,
               ROW_NUMBER() OVER (PARTITION BY column_name ORDER BY column_name) AS row_num
            FROM table_name;

9. Recursive Query:

Performs a recursive query.

            WITH RECURSIVE subquery_name AS (
              SELECT column_name
              FROM table_name
              WHERE condition
              UNION ALL
              SELECT column_name
              FROM table_name
              WHERE condition
            )
            SELECT * FROM subquery_name;

10. Dynamic SQL:

Executes dynamically built SQL statements.

            EXECUTE immediate 'SELECT * FROM ' || table_name;

Standard Code Snippets

Basic Level

Create a Table

Create a new table in the database.

                                CREATE TABLE employees (
                                    id INT PRIMARY KEY,
                                    name VARCHAR(100),
                                    age INT,
                                    department VARCHAR(50)
                                );
Insert Data into a Table

Insert new data into an existing table.

                                INSERT INTO employees (id, name, age, department)
                                VALUES (1, 'Sahil', 22, 'HR');
Select All Data

Retrieve all data from a table.

                                SELECT * FROM employees;
Update a Record

Update an existing record in a table.

                                UPDATE employees
                                SET age = 31
                                WHERE id = 1;
Delete a Record

Delete a record from a table.

                                DELETE FROM employees
                                WHERE id = 1;
Filter Data Using WHERE

Filter data based on a condition.

                                SELECT * FROM employees
                                WHERE age > 25;
Add a New Column to a Table

Add a new column to an existing table.

                                ALTER TABLE employees
                                ADD email VARCHAR(100);
Create a Simple View

Create a simple view of a table.

                                CREATE VIEW employee_view AS
                                SELECT name, department FROM employees;
Drop a Table

Drop an existing table.

                                DROP TABLE employees;
Get the Maximum Value

Get the maximum value of a column.

                                SELECT MAX(age) FROM employees;

Intermediate Level

Join Two Tables

Join two tables based on a common column.

                                SELECT e.name, d.department_name
                                FROM employees e
                                INNER JOIN departments d ON e.department = d.department_id;
Group Data and Count

Group data and count the number of rows.

                                SELECT department, COUNT(*)
                                FROM employees
                                GROUP BY department;
Create an Index

Create an index on a column for faster querying.

                                CREATE INDEX idx_name ON employees(name);
Create a Foreign Key

Create a foreign key constraint between two tables.

                                ALTER TABLE employees
                                ADD CONSTRAINT fk_department FOREIGN KEY (department)
                                REFERENCES departments(department_id);
Use CASE in SELECT

Use a CASE statement in a SELECT query.

                                SELECT name, age,
                                CASE
                                    WHEN age < 30 THEN 'Junior'
                                    ELSE 'Senior'
                                END AS level
                                FROM employees;
Use a Subquery

Use a subquery to filter data.

                                SELECT name, age
                                FROM employees
                                WHERE age > (SELECT AVG(age) FROM employees);
Calculate the Total with SUM

Calculate the total sum of a column.

                                SELECT SUM(salary) FROM employees;
Create a Trigger

Create a trigger to perform an action on data modification.

                                CREATE TRIGGER update_timestamp
                                BEFORE UPDATE ON employees
                                FOR EACH ROW
                                SET NEW.updated_at = NOW();
Use DISTINCT to Remove Duplicates

Use the DISTINCT keyword to remove duplicate rows.

                                SELECT DISTINCT department FROM employees;
Select Top N Rows

Select the top N rows from a table.

                                SELECT * FROM employees
                                ORDER BY salary DESC
                                LIMIT 5;

Expert Level

Recursive Query for Hierarchical Data

Use a recursive query to retrieve hierarchical data.

                                WITH RECURSIVE employee_hierarchy AS (
                                    SELECT id, name, manager_id
                                    FROM employees
                                    WHERE manager_id IS NULL
                                    UNION ALL
                                    SELECT e.id, e.name, e.manager_id
                                    FROM employees e
                                    INNER JOIN employee_hierarchy eh ON e.manager_id = eh.id
                                )
                                SELECT * FROM employee_hierarchy;
Pivot Data

Pivot data to transform rows into columns.

                                SELECT *
                                FROM (SELECT department, salary FROM employees) AS source
                                PIVOT (AVG(salary) FOR department IN ('HR', 'IT', 'Sales')) AS pvt;
Dynamic SQL Execution

Execute dynamic SQL queries.

                                SET @sql = 'SELECT * FROM ' + @table_name;
                                PREPARE stmt FROM @sql;
                                EXECUTE stmt;
                                DEALLOCATE PREPARE stmt;
Window Function with Ranking

Use a window function to rank data.

                                SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rank
                                FROM employees;
Create a Stored Procedure

Create a stored procedure to perform a complex operation.

                                CREATE PROCEDURE update_salary(IN employee_id INT, IN salary_increase DECIMAL)
                                BEGIN
                                    UPDATE employees
                                    SET salary = salary + salary_increase
                                    WHERE id = employee_id;
                                END;
Using CTE with Aggregation

Use a common table expression (CTE) with aggregation.

                                WITH sales_data AS (
                                    SELECT department, SUM(sales) AS total_sales
                                    FROM sales
                                    GROUP BY department
                                )
                                SELECT * FROM sales_data WHERE total_sales > 100000;
Using JSON Data in SQL

Use JSON data in SQL queries.

                                SELECT name, json_data->>'$.key' AS key_value
                                FROM employees
                                WHERE json_data IS NOT NULL;
Partitioning Table Data

Partition table data for better performance.

                                CREATE TABLE employees_partitioned (
                                    id INT, name VARCHAR(100), salary DECIMAL
                                )
                                PARTITION BY RANGE (salary) (
                                    PARTITION p0 VALUES LESS THAN (30000),
                                    PARTITION p1 VALUES LESS THAN (60000),
                                    PARTITION p2 VALUES LESS THAN (90000)
                                );
Error Handling in Transactions

Handle errors in transactions.

                                START TRANSACTION;
                                BEGIN TRY
                                    INSERT INTO employees (id, name, salary) VALUES (1, 'John', 50000);
                                    COMMIT;
                                EXCEPTION
                                    ROLLBACK;
                                END TRY;
