JavaScript Resources

Language Snapshot

Overview

  • Definition: JavaScript is a versatile, high-level programming language primarily used for enhancing interactivity on web pages.
  • Creator: Brendan Eich
  • Year Created: 1995
  • Purpose: JavaScript enables developers to implement complex features on web pages, making it an essential part of web development alongside HTML and CSS.

Key Features

  • Dynamic Typing: JavaScript is dynamically typed, meaning variable types are determined at runtime, allowing for more flexibility in coding.
  • Prototype-Based Object Orientation: Instead of traditional class-based inheritance, JavaScript uses prototypes, enabling objects to inherit directly from other objects.
  • First-Class Functions: Functions in JavaScript are treated as first-class citizens, meaning they can be assigned to variables, passed as arguments, and returned from other functions.
  • Event-Driven: JavaScript is primarily event-driven, allowing developers to create interactive user interfaces that respond to user actions.
  • Asynchronous Programming: With features like callbacks, promises, and async/await, JavaScript supports non-blocking code execution, making it ideal for handling I/O operations.

Common Use Cases

  • Web Development: Enhancing the user experience on websites through interactivity and dynamic content.
  • Server-Side Development: With Node.js, JavaScript can also be used for server-side programming.
  • Mobile App Development: Frameworks like React Native allow developers to build mobile applications using JavaScript.
  • Game Development: JavaScript can be used to create browser-based games.

Cheat Sheet

Basic Level

1. Variables:

Used to store data values.

            let variableName = value; // let is block-scoped
            const constantName = value; // const is for constants

2. Data Types:

Different types of data that can be stored in variables.

            String: "Sahil"
            Number: 09
            Boolean: true or false
            Null: null
            Undefined: undefined
            Symbol: Symbol('description')

3. Functions:

A block of code designed to perform a particular task.

            function functionName(parameters) {
                // code to be executed
            }

4. If Statement:

Executes a block of code if a specified condition is true.

            if (condition) {
                // code to be executed
            }

5. Loops:

Executes a block of code a number of times.

            for (let i = 0; i < 5; i++) {
                // code to be executed
            }

Intermediate Level

1. Arrays:

A collection of elements.

            let arrayName = [element1, element2, element3];

2. Objects:

A collection of key-value pairs.

            let objectName = {
                key1: value1,
                key2: value2
            };

3. Arrow Functions:

A concise way to write functions.

            const functionName = (parameters) => {
                // code to be executed
            };

4. Promise:

An object that represents the eventual completion (or failure) of an asynchronous operation.

            const myPromise = new Promise((resolve, reject) => {
                // asynchronous code
            });

5. Try /Catch:

Used to handle exceptions.

            try {
                // code that may throw an error
            } catch (error) {
                // code to handle the error
            }

Expert Level

1. Closures:

A function that retains access to its lexical scope, even when the function is executed outside that scope.

            function outerFunction() {
                let outerVariable = 'I am outside!';
                function innerFunction() {
                    console.log(outerVariable);
                }
                return innerFunction;
            }

2. IIFE (Immediately Invoked Function Expression):

A function that runs as soon as it is defined.

            (function() {
                // code to be executed
            })();

3. Async/Await:

A syntactic sugar for working with Promises, making asynchronous code look synchronous.

            async function myFunction() {
                const result = await myPromise;
            }

4. Spread Operator:

Allows an iterable such as an array or string to be expanded in places where zero or more arguments are expected.

            let newArray = [...array1, ...array2];

5. Destructuring Assignment:

A syntax that makes it possible to unpack values from arrays or properties from objects into distinct variables.

            const { key1, key2 } = objectName;

Expert Level

Throttling a Function

Throttling a function to prevent excessive calls.

                                function throttle(func, limit) {
                                    let lastFunc;
                                    let lastRan;
            
                                    return function() {
                                        const context = this;
                                        const args = arguments;
            
                                        if (!lastRan) {
                                            func.apply(context, args);
                                            lastRan = Date.now();
                                        } else {
                                            clearTimeout(lastFunc);
                                            lastFunc = setTimeout(function() {
                                                if ((Date.now() - lastRan) >= limit) {
                                                    func.apply(context, args);
                                                    lastRan = Date.now();
                                                }
                                            }, limit - (Date.now() - lastRan));
                                        }
                                    };
                                }

Creating a Debounce Function

Creating a debounce function to prevent excessive calls.

                                function debounce(func, wait) {
                                    let timeout;
                                    return function(...args) {
                                        const context = this;
                                        clearTimeout(timeout);
                                        timeout = setTimeout(() => func.apply(context, args), wait);
                                    };
                                }
Memoization

Using memoization to optimize function calls.

                                function memoize(fn) {
                                    const cache = {};
                                    return function(...args) {
                                        const key = JSON.stringify(args);
                                        if (cache[key]) {
                                            return cache[key];
                                        }
                                        const result = fn(...args);
                                        cache[key] = result;
                                        return result;
                                    };
                                }
